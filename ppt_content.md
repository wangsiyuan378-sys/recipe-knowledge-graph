**关系类型:**

- RECOMMENDS: 食谱推荐其他食谱（reason）

## 幻灯片7: 知识图谱模型

**实体类型:**
- � Recipe（食谱）- 菜品信息
- 🥬 Ingredient（原材料）- 食材信息
- � Step（步骤）- 制作步骤

**关系类型:**
- NEEDS（需要）：Recipe → Ingredient
- HAS_STEP（包含步骤）：Recipe → Step
- RECOMMENDS（推荐）：Recipe → Recipe

**示例三元组:**
```
(西红柿炒鸡蛋) -[:NEEDS]-> (鸡蛋)
(西红柿炒鸡蛋) -[:NEEDS]-> (西红柿)
(西红柿炒鸡蛋) -[:HAS_STEP]-> (步骤1)
(宫保鸡丁) -[:RECOMMENDS]-> (鱼香肉丝)
```

**知识图谱结构图:**
```
        ┌──────────────────┐
        │   西红柿炒鸡蛋    │
        └────────┬─────────┘
                 │ NEEDS
        ┌────────┼────────┐
        ▼        ▼        ▼
   ┌────────┐ ┌────────┐ ┌────────┐
   │  鸡蛋   │ │ 西红柿  │ │   葱   │
   └────────┘ └────────┘ └────────┘
```

## 幻灯片9: 10个Cypher查询示例（1-5）

| 编号 | 查询功能 | Cypher语句 |
|------|---------|-----------|
| 1 | 查询所有川菜 | `MATCH (r:Recipe {category: '川菜'}) RETURN r.name, r.difficulty` |
| 2 | 查询菜品原材料 | `MATCH (r:Recipe {name: '宫保鸡丁'})-[rel:NEEDS]->(i) RETURN i.name, rel.quantity` |
| 3 | 查询制作步骤 | `MATCH (r:Recipe {name: '西红柿炒鸡蛋'})-[:HAS_STEP]->(s) RETURN s.step_number, s.description` |
| 4 | 查询简单菜品 | `MATCH (r:Recipe {difficulty: '简单'}) RETURN r.name, r.cook_time LIMIT 10` |
| 5 | 查询推荐菜品 | `MATCH (r:Recipe {name: '宫保鸡丁'})-[:RECOMMENDS]->(rec) RETURN rec.name` |

## 幻灯片11: 智能推荐功能

**推荐算法逻辑:**
1. 用户输入现有食材（如"鸡蛋,西红柿"）
2. 系统从知识图谱检索包含这些食材的菜品
3. 计算匹配度 = 匹配食材数 / 总食材数
4. 按匹配度排序，返回推荐表格

**推荐结果展示:**

| 菜品 | 菜系 | 难度 | 烹饪时间 | 匹配食材 | 匹配度 |
|------|------|------|---------|---------|--------|
| 西红柿炒鸡蛋 | 家常菜 | 简单 | 15分钟 | 鸡蛋,西红柿 | 100% |
| 西红柿鸡蛋汤 | 家常菜 | 简单 | 10分钟 | 鸡蛋,西红柿 | 66% |
| 蛋炒饭 | 家常菜 | 简单 | 10分钟 | 鸡蛋 | 33% |

**算法优势:**
- 基于知识图谱的真实数据，推荐准确
- 显示已有食材和缺少食材，方便用户决策
- 支持多食材组合查询，灵活实用

## 幻灯片12: AI智能问答（RAG）

**RAG工作流程:**
```
用户提问
    │
    ▼
提取关键词（如"西红柿炒鸡蛋"）
    │
    ▼
知识图谱检索（Cypher查询）
    │
    ▼
获取菜品原材料、步骤等上下文
    │
    ▼
组合问题+上下文 → 发送给DeepSeek API
    │
    ▼
AI生成基于知识图谱的回答
```

**DeepSeek API集成:**
```python
client = OpenAI(

    base_url="https://api.deepseek.com"
```

## 幻灯片18: 遇到的问题与解决

**问题1: Neo4j连接失败**
- 现象: `Failed to DNS resolve address`
- 原因: 实例未启动或凭证错误
- 解决: 检查实例状态，更新连接信息

**问题2: Cypher语法错误**
- 现象: 参数名不匹配
- 原因: 查询参数与传入参数不一致
- 解决: 统一参数名称

**问题3: Git推送失败**
- 现象: `src refspec main does not match any`
- 原因: 本地分支为master
- 解决: `git branch -m master main`

**问题4: Streamlit环境变量格式错误**
- 现象: `Invalid format: please enter valid TOML`
- 原因: 环境变量值未加引号
- 解决: 添加双引号

**问题5: 关系数量为0**
- 现象: 统计显示关系为0
- 原因: 数据导入不完整
- 解决: 重新运行导入脚本
