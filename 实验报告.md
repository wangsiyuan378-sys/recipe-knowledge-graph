# 基于知识图谱的智能食谱推荐系统实验报告

---

## 摘要

本实验项目基于知识图谱技术，构建了一个智能食谱推荐系统。系统采用 Neo4j Aura 云端图数据库存储食谱数据，通过 Streamlit 框架构建 Web 用户界面，并集成 DeepSeek 大语言模型实现 RAG（检索增强生成）智能问答功能。用户可以通过本系统搜索菜谱、查看食材清单和制作步骤、根据现有食材智能推荐菜品，以及与 AI 进行对话获取烹饪建议。

**关键词**：知识图谱、Neo4j、Streamlit、DeepSeek、RAG、食谱推荐

---

## 一、实验目的

1. **掌握知识图谱构建技术**：学习使用 Neo4j 图数据库设计和构建知识图谱，掌握实体、关系、属性的设计与实现方法。

2. **掌握 Cypher 查询语言**：熟练运用 Cypher 查询语言进行复杂的关系查询和数据检索。

3. **掌握 Web 应用开发技术**：使用 Streamlit 框架快速构建数据可视化和交互式 Web 应用。

4. **掌握大语言模型集成技术**：学习将大语言模型（DeepSeek）与知识图谱结合，实现 RAG 智能问答系统。

5. **掌握云端部署技术**：学习将 Web 应用部署到 Streamlit Community Cloud，实现可公开访问的在线服务。

## 三、实验内容与步骤

### 3.1 知识图谱设计

#### 3.1.1 需求分析

本项目旨在构建一个食谱知识图谱系统，用户需求包括：

1. **菜品搜索功能**：根据菜名或关键词搜索菜品信息
2. **菜品详情展示**：展示菜品的原材料清单、制作步骤、烹饪时间等
3. **原材料查询**：根据原材料查询使用该材料的菜品
4. **智能推荐**：根据用户输入的现有食材推荐可制作的菜品
5. **AI 智能问答**：基于知识图谱数据与大模型结合的智能问答

#### 3.1.2 数据模型设计

本系统设计了以下实体和关系：

**实体类型**：

| 实体 | 属性 | 说明 |
|------|------|------|
| Recipe（食谱） | recipe_id, name, difficulty, cook_time, category, description, taste | 存储菜品基本信息 |
| Ingredient（原材料） | ingredient_id, name, category, unit, calories_per_100g | 存储食材信息 |
| Step（步骤） | step_number, description | 存储制作步骤 |

**关系类型**：

| 关系 | 起点→终点 | 属性 | 说明 |
|------|----------|------|------|
| NEEDS | Recipe→Ingredient | quantity, note | 表示食谱需要的原材料及用量 |
| HAS_STEP | Recipe→Step | - | 表示食谱的制作步骤 |
| RECOMMENDS | Recipe→Recipe | reason | 表示菜品推荐关系 |

#### 3.4.4 智能推荐功能

**实现逻辑**：
1. 用户输入现有食材（如"鸡蛋,西红柿,葱"）
2. 系统从知识图谱中检索所有包含这些食材的菜品
3. 计算每个菜品的匹配度（匹配食材数/总食材数）
4. 按匹配度排序，返回推荐结果

**关键代码**：
```python
def recommend_by_ingredients(driver, ingredients_list):
    all_recipes = get_all_recipes(driver)
    matching_recipes = []
    
    for recipe in all_recipes:
        recipe_details = get_recipe_details(driver, recipe)
        recipe_ingredients = [ing['原材料'] for ing in recipe_details[1]]
        
        matched = set(ingredients_list) & set(recipe_ingredients)
        if matched:
            match_rate = len(matched) / len(recipe_ingredients)
            matching_recipes.append({
                '菜品': recipe,
                '匹配食材': ', '.join(matched),
                '匹配度': f"{match_rate:.1%}"
            })
    
    return sorted(matching_recipes, key=lambda x: float(x['匹配度'].replace('%','')), reverse=True)
```

#### 3.4.5 AI 智能问答功能（RAG）

**RAG 工作流程**：
1. 用户输入问题（如"西红柿炒鸡蛋怎么做？"）
2. 系统从知识图谱中检索相关食谱数据
3. 将问题与检索到的上下文组合，发送给 DeepSeek API
4. AI 结合知识图谱数据生成回答

## 五、对比分析

### 5.1 纯大模型问答 vs 知识图谱+大模型问答

#### 5.1.1 实验设计

为验证知识图谱在智能问答中的价值，设计了5组对比实验：

| 问题编号 | 问题内容 |
|---------|---------|
| Q1 | 西红柿炒鸡蛋怎么做？ |
| Q2 | 宫保鸡丁需要哪些食材？ |
| Q3 | 教我做一道简单的家常菜 |
| Q4 | 红烧肉的做法步骤 |
| Q5 | 如何做出一道好吃的糖醋排骨？ |

#### 5.1.2 实验结果

| 问题 | 纯大模型回答 | 知识图谱+大模型回答 | 优势分析 |
|------|------------|-------------------|---------|
| Q1 | 可能包含常见做法，但不一定准确 | 基于数据库中的标准菜谱回答，步骤准确 | ✅ 准确度更高 |
| Q2 | 可能列举常见食材，但可能遗漏 | 基于数据库中的实际食材清单回答 | ✅ 信息更完整 |
| Q3 | 可能推荐常见菜品，但缺乏具体信息 | 可以推荐具体菜品并提供详细信息 | ✅ 推荐更具体 |
| Q4 | 可能包含正确步骤，但不一定与数据库一致 | 基于实际菜谱数据回答 | ✅ 一致性更好 |
| Q5 | 可能包含正确做法，但可能不够详细 | 可以提供精确的食材用量和步骤 | ✅ 细节更丰富 |

#### 5.1.3 结论

知识图谱+大模型的 RAG 模式相比纯大模型问答具有以下优势：

1. **回答准确性更高**：基于实际数据库中的菜谱信息，避免大模型的幻觉问题
2. **信息完整性更好**：可以提供精确的食材用量、烹饪时间等信息
3. **一致性更强**：回答的内容与实际菜谱保持一致
4. **可追溯性强**：回答基于的知识来源明确可信

## 六、问题与解决

### 6.1 Neo4j Aura 连接问题

**问题描述**：
```
neo4j.exceptions.ServiceUnavailable: Failed to DNS resolve address
```

**原因分析**：
- Neo4j Aura 实例未启动
- 网络连接问题
- 连接凭证错误

**解决方案**：
1. 登录 Neo4j Aura 控制台检查实例状态
2. 如实例暂停，点击启动按钮
3. 确认连接凭证（URI、用户名、密码）正确

### 6.2 Cypher 语法错误

**问题描述**：
```
neo4j.exceptions.CypherSyntaxError: Parameters not valid
```

**原因分析**：
- 查询语句中参数名不匹配
- 参数类型错误

**解决方案**：
确保查询语句中的参数名与传入参数一致：
```python
# 错误写法
session.run("MATCH (r:Recipe {name: $name})", {'ingredient_name': recipe_name})

# 正确写法
session.run("MATCH (r:Recipe {name: $name})", {'name': recipe_name})
```

### 6.3 Git 推送问题

**问题描述**：
```
error: src refspec main does not match any
```

**原因分析**：
- 本地分支名为 `master`，远程分支名为 `main`

**解决方案**：
```bash
git branch -m master main
git push -u origin main
```

### 6.4 Streamlit 环境变量格式错误

**问题描述**：
```
Invalid format: please enter valid TOML

**原因分析**：

在 Streamlit Cloud 中设置环境变量时，确保格式正确：

```

### 6.5 关系数量为0问题

**问题描述**：
