### 3.1 知识图谱设计

#### 3.1.1 实体设计

本系统设计了以下实体类型：

| 实体类型 | 说明 | 属性 |
|---------|------|------|
| Recipe | 食谱 | recipe_id, name, difficulty, cook_time, category, description, taste |
| Ingredient | 原材料 | ingredient_id, name, category, unit, calories_per_100g |
| Step | 制作步骤 | step_number, description, duration |

#### 3.1.2 关系设计

| 关系类型 | 起始节点 | 目标节点 | 属性 | 说明 |
|---------|---------|---------|------|------|
| NEEDS | Recipe | Ingredient | quantity, note | 食谱需要原材料 |
| HAS_STEP | Recipe | Step | - | 食谱包含制作步骤 |
| RECOMMENDS | Recipe | Recipe | reason | 食谱推荐关系 |

#### 3.1.3 知识图谱结构图

```
┌─────────────┐
│   Recipe    │
│  (食谱)     │
└──────┬──────┘
       │
       ├── NEEDS ──────────→ ┌─────────────┐
       │   (需要)           │ Ingredient  │
       │                    │  (原材料)   │
       │                    └─────────────┘
       │
       ├── HAS_STEP ───────→ ┌─────────────┐
       │   (包含步骤)        │    Step     │
       │                    │  (步骤)     │
       │                    └─────────────┘
       │
       └── RECOMMENDS ──────→ ┌─────────────┐
           (推荐)            │   Recipe    │
                             │  (其他食谱) │
                             └─────────────┘
```

```python
def search_recipes(driver, keyword):

        MATCH (r:Recipe)
```

**2. 菜品详情模块**

```python
def get_recipe_details(driver, recipe_name):
    # 获取食谱基本信息
    recipe_query = """
        MATCH (r:Recipe {name: $name})
        RETURN r.name AS 菜名, r.category AS 菜系, 
               r.difficulty AS 难度, r.cook_time AS 烹饪时间
    """
    # 获取原材料
    ingredients_query = """
        MATCH (r:Recipe {name: $name})-[rel]->(i:Ingredient)
        RETURN i.name AS 原材料, rel.quantity AS 用量, rel.note AS 备注
    """
    # 获取制作步骤
    steps_query = """
        MATCH (r:Recipe {name: $name})-[:HAS_STEP]->(s:Step)
        RETURN s.step_number AS 步骤, s.description AS 说明
        ORDER BY s.step_number
    """
```

```python
def get_recommendations(driver, recipe_name=None):
```

### 6.2 Cypher语法错误

**问题描述：**
```
neo4j.exceptions.CypherSyntaxError: Parameter name mismatch
```

**原因分析：**
- 查询中使用`$name`但传入参数为`ingredient_name`
- 参数名不一致

**解决方案：**
```python
# 错误写法
query = "MATCH (i:Ingredient {name: $name}) ..."
params = {'ingredient_name': '鸡蛋'}

# 正确写法
query = "MATCH (i:Ingredient {name: $name}) ..."
params = {'name': '鸡蛋'}
```

- 分支名称不一致

**解决方案：**
```bash
git branch -m master main
git push -u origin main
```

### 6.4 Streamlit环境变量格式错误

**问题描述：**

```
Invalid format: please enter valid TOML
```

**原因分析：**
- 环境变量值未加引号

**解决方案：**
```
# 错误格式
NEO4J_URI = neo4j+s://xxx.databases.neo4j.io

```

### 6.5 数组索引越界

**问题描述：**
```
IndexError: list index out of range
```

**原因分析：**
- `st.columns(2)` 创建2列，但循环访问3个元素

**解决方案：**
```python
# 错误写法
cols = st.columns(2)
for idx, question in enumerate(questions):
    cols[idx].button(question)  # idx=2时越界

# 正确写法
cols = st.columns(3)
for idx, question in enumerate(questions):
    cols[idx % 3].button(question)
```
